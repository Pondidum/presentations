  • git is a 3rd generation
  • 3rd gen are better because: offline repository, branching a key feature, decentralised
  • 3rd gens are: mercurial, git, bazaar
  • git was picked as
    • its the most popular/versatile of the 3rd generations
    • tfs/everything supports it
    • easy migration path in the future
  • you can interact with git/tfs via
    • visual studio
    • command line

Demo!

  • commit little and often
  • eg adding a few references to a project and adding a new file should be separate
  • 3 standard branches:
    • master - no committing to this!
    • development - do your work here
    • current release - bug fixes
  • use your own branches (locally)
    • experiment
    • fix bugs
    • merge back to parent branch (either development or release)
merging your own local branches is fine dont merge master/dev/release to/from master/dev/release without asking!
