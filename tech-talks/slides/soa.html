What is it?
  • It's an alternative to Monolithic Architecture
  • It helps keep everything separated
  • Current vogue is for MicroServices
  • we split things into separate focused services
  • how to communicate between them?
  • HTTP?! The answer to all questions...right?
  • Well...no. exchanged references for http calls
  • Still tightly coupled.
  • Everything needs to know where its endpoints are
  • what happens if you want to publish a piece of data TO many unknown services?
  • Who cares about your published info is not your concern
  • Luckily, messagebusses/queues
  • push things you do to the bus
  • listen to any message which interests you
  • Great, I can just do that with a sql table
No

  • New pages are constantly allocated and deallocated in the buffer pool
  • There are solutions
  • But if Sql Server is your Hammer, all problems are Nails.

  • Why not use a tool designed for this?
  • zeromq is very low level, but has insane performance
  • rabbit is higher level, but still very very good performance
  • queues can be set up in many different ways, and for different durability guarantees
