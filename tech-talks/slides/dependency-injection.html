Service Oriented Architecture


What is it?
  • Rather than creating instances inside your class, have them passe in
  • It is the D in SOLID
  • Simplification of classes (no need to setup your dependencies)
  • SRP (not your job to setup dependencies)
  • Testability (pass in fakes)
  • Its easy (even without a container)
  • Constructor Injection (implies required dependency)
  • Property Injection (implies optional dependency)
  • Interface Injection (I know how it works, but not seen any real use of it)

Constructor Injection


  public class StreamReader {
    private readonly Stream _stream;

    public StreamReader(Stream stream) {
      _stream = stream;
    }

    public void ReadLine() {
      var sb = new StringBuilder();
      var current = '';

      while ((current = _stream.ReadChar()) != null) {
        if (current == "\r" || current == "\n") break;
        sb.Append(current);
      }

      return sb.ToString();
    }
  }
  • Should be your most used
  • Example requires you to pass in the stream to read
  • But creates its own StringBuilder...

Property Injection


  public class Cache<TKey, TValue> {

    public Func<TKey, TValue> OnMissing { get; set; }

    private readonly Dictionary<TKey, TValue> _contents;

    public Cache() {
      _contents = new Dictionary<TKey, TValue>();
      OnMissing = key => { };
    }

    public TValue Get(TKey key) {
      if (_contents.ContainsKey(key) == false) {
        _contents[key] = OnMissing.Invoke(key);
      }

      return _contents.GetOrDefault(key);
    }
  }
  • Has occasional use
  • Example has its own default action (do nothing)
  • Can push in your own function to the OnMissing property

  var container = new Container(c => c.Scan(a => {
    a.TheCallingAssembly();

  }));

  public class BasicRegistry : Registry {

    public BasicRegistry() {

      For<IBrandFactory>().Singleton();
      For<ISettings>().Singleton();

      Scan(a => {
        a.AddAllTypesOf<IEnhancement>();
      });
    }
  }
  • First we tell Structuremap to use the calling assembly (one it is being executed in)
  • `WithDefaultConventions` => this adds standard mapping of "ISomeClass => SomeClass"
  • `LookForRegistries` => Add in configuration found in Registries
  • Registries allow you to split your configuration (by featureset usually, eg one for email things, one for ui hookup etc)
  • You should not reference the container anywhere other than startup...

  public class IndexController : ApiController {

    private readonly SparkResponseFactory _factory;
    private readonly WidgetCollection _widgets;
    private readonly IndexViewModelBuilder _builder;

    public IndexController(SparkResponseFactory factory, WidgetCollection widgets, IndexViewModelBuilder builder) {
      _factory = factory;
      _widgets = widgets;
      _builder = builder;
    }

    public HttpResponseMessage GetIndex() {
      return _factory.From(new IndexViewModel {
        Definitions = _widgets.Select(_builder.FromWidget)
      });
    }
  }
